Getting Started¶

This chapter covers how to establish connection between the robot and the user’s workstation PC, and run some examples to verify system functionalities.

Establish connection¶

The following steps show how to physically connect user’s workstation PC to the robot:

1. Make sure the system is powered OFF and the robot arm is mounted on a stable base. The robot must be securely mounted onto a steady base and cannot topple over even when the robot is moving at high speeds with sudden stops.

2. Using an Ethernet cable, connect the Ethernet port on the workstation PC to the Ethernet port on the control box that is marked as LAN. Ensure that connectors are firmly attached.

Important

Going through a router or an ethernet switch can result in reduced control performance due to extra network latency.

  1. With the system powered OFF, push the mode selection switch on the motion bar to upper position (i.e. auto mode).

    Note

    Refer to the user manual for more details about auto mode and manual mode. Here we’ll need to use auto mode for RDK.

  2. Check that the emergency stop (E-stop) on the motion bar is pressed down.

  3. Power ON the system using the power switch located on the control box, then wait until booting is complete. The LED rings on the robot will turn solid blue when the robot is ready. This process may take up to 60 seconds.

    Caution

    Blinking LED rings on the robot indicate a fault occurred. Please refer to the Troubleshooting chapter for more details.

  4. The default IP address of the robot is 192.168.2.100. By this time the connection should already have been established, so check the connection by pinging the robot from the workstation PC:

    ping 192.168.2.100

    The time value from the ping command denotes the round trip time and this value should be well below 1 ms

    Note

    The workstation PC must be in the same subnet (i.e. with IP address 192.168.2.x) with the robot in order to connect. However, the user does NOT need to configure the workstation PC’s IP address manually, because the robot has its own DHCP server built in. So when the robot finishes booting with the workstation PC is connected, the DHCP server will automatically assign an IP address in the same subnet to the workstation PC.

Setup environment on workstation PC¶

Update C++ standard headers¶

Note

Skip this step if a) the workstation PC is running Ubuntu 20.04 as this update is only needed by Ubuntu 18.04, OR b) you will only use RDK’s Python interface but not the C++ interface.

Static library libFlexivRdk.a for RDK’s C++ interface is compiled by a Ubuntu 20.04 machine, which uses newer C++ standard than that of Ubuntu 18.04. Thus, for users with Ubuntu 18.04, they need to update the C++ standard headers installed in the workstation PC in order to successfully link to the static library. Use the Terminal commands below to do the update:

sudo add-apt-repository ppa:ubuntu-toolchain-r/test
sudo apt update
sudo apt install libstdc++6

To make sure the update is successful, check the version of libstdc++6 using:

sudo apt install libstdc++6

Look for the message libstdc++6 is already the newest version, followed by the actual version number, which should be greater than 10.0.

Update Python3 version¶

Note

Skip this step if you will only use RDK’s C++ interface but not the Python interface.

Dynamic library flexivrdk.so for RDK’s Python interface is compiled by a Ubuntu 20.04 machine with Python 3.8. Thus, users will need Python 3.8 in order to successfully import this library and execute their Python script.

Note

As long as Python 3.8 is used, both Ubuntu 18.04 and 20.04 will work.

Now check if the Python version installed on the workstation PC has already met the requirement:

python3 --version

By default, Ubuntu 18.04 ships with Python 3.6.x and Ubuntu 20.04 ships with Python 3.8.x. If the version is not 3.8.x, a standalone Python 3.8 can be installed from Deadsnakes PPA without affecting the existing Python3 installation:

sudo apt update
sudo apt install software-properties-common
sudo add-apt-repository ppa:deadsnakes/ppa

Press Enter to confirm adding the PPA, then install the standalone Python 3.8:

sudo apt install python3.8

Verify that the installation was successful:

python3.8 --version

The output should say Python 3.8.x. Later use python3.8 command to explicitly invoke Python 3.8.

Warning

Using Python version other than 3.8.x with flexivrdk.so may result in runtime errors like segmentation fault.

Obtain RDK library files¶

The Flexiv RDK library consists of headers, static library, examples, URDF, mesh files, etc. These are all hosted in a public GitHub repository at https://github.com/flexivrobotics/flexiv_rdk/.

You can now Git clone this repository or directly download it as a zip file to the workstation PC.

Run example programs¶

Now that the connection is established, we’ll need to enable the robot and verify the whole setup by running the display_robot_states example, which will enable the robot then continuously get and print the current states of the robot while the robot holds itself at the current pose.

Preparation¶

  1. Make sure the robot has already finished booting and the connection between the workstation PC and the robot server is successfully established according to the instructions above. Then the robot server should’ve assigned an IP address to the workstation PC, which can be obtained using commands:

    sudo apt install net-tools
ifconfig

    You should see that one of the workstation PC’s network interfaces has been assigned an IP address within the same subnet as the robot server, for example, 192.168.2.200.

  2. Release (pull up) the emergency stop (E-stop) button on the motion bar. The robot will have no action because an enabling method must be called to actually enable the robot.

C++ interface¶

  1. Configure CMake and compile all C++ example programs:

    cd flexiv_rdk
mkdir build && cd build
cmake ..
make -j4

  2. Run the compiled example program display_robot_states, which will first call the aforementioned method to enable the robot, then hold the robot while printing out all robot states:

    cd flexiv_rdk/build/example
sudo ./display_robot_states <robot_ip> <local_ip>

    sudo is required to grant root privileges to the integrated scheduler, and <local_ip> is the workstation PC’s IP address obtained during preparation, for example:

    sudo ./display_robot_states 192.168.2.100 192.168.2.200

  3. The robot will be enabled and release the joint brakes, which will make some audible crispy sound. After a few seconds, when the enabling process is done and the robot becomes operational, the integrated scheduler will start a real-time periodic loop running user tasks at 1kHz. Python interface¶

  1. Run the example Python script display_robot_states.py, which will first call the aforementioned method to enable the robot, then hold the robot while printing out all robot states:

    cd flexiv_rdk/example_py
sudo python3 display_robot_states.py <robot_ip> <local_ip>

    sudo is required to grant root privileges to the integrated scheduler, and <local_ip> is the workstation PC’s IP address obtained during preparation, for example:

    sudo python3 display_robot_states.py 192.168.2.100 192.168.2.200

    Note

    Replace python3 with python3.8 if you’ve done the Python version update using instructions provided in Update Python3 version section.

  2. The robot will be enabled and release the joint brakes, which will make some audible crispy sound. After a few seconds, when the enabling process is done and the robot becomes operational, a non-real-time periodic loop constructed using while loop and sleep() method will start. In this example, the periodic loop mainly obtains and prints the robot states.

  3. To close the running example program, simply press CTRL+C.

Note

When the RDK user program is terminated or the connection is lost, the robot will automatically decelerate to a halt and hold its position. The robot server supports on-the-fly reconnection, so you can terminate a user program and start a new one without doing anything to the robot server.

The next chapter will provide more details on the RDK library, and you can also check out the other examples to get familiar with RDK APIs and how to use them.
